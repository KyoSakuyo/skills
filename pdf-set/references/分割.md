## 输入/输出
- 输入：`合并结果/0.粗全文.md`。
- 输出：按分割结果生成多个文件，命名为 `从1计数的序号.标题名.md`，写入 `合并结果/`。（因为已经有0开头的`合并结果/0.粗全文.md`，所以无论你遇到什么情况，都用1开始计数。）

## 脚本参考

- 使用 `scripts/split.py` 完成分割。
- 默认按当前工作目录推导路径：
  - 输入：`<当前目录>/合并结果/0.粗全文.md`
  - 输出：`<当前目录>/合并结果/`
- 可选参数：
  - `--base-dir` 指定书籍目录
  - `--input-file` 指定完整输入文件路径
  - `--input-file-from` 使用 UTF-8 文本文件提供输入文件路径（取首个非空行）
  - `--output-dir` 指定输出目录

示例：

```bash
python .agent/skills/pdf-set/scripts/split.py --base-dir "C:\path\to\book"
```

## 阶段 1：确定分割依据
1. 优先使用 Markdown **一级标题**（`# `）作为分割点。
2. 二级及以下标题 **不得** 用于分割。
3. 若用户提供了目录名列表：
   - 若文中存在这些目录名，按指定目录名命名；
   - 若不存在一级标题，则需自行判断合适分割点，并按目录名命名。

## 阶段 2：执行分割

- 仅分割与命名，不做任何排版、清理、纠错或格式调整。
- 保持原文内容完整，按顺序输出到各自文件。

## 阶段 3：输出检查
- 所有分割文件位于 `合并结果/`。
- 文件名格式严格为 `1.标题名.md`、`2.标题名.md`……
- 分割顺序与原文顺序一致。
