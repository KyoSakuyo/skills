CRITICAL: 按顺序执行以下阶段，勿跳步。

## 输入/输出
- 输入：`typeset-result/` 中的文件，按文件名前序号排序。
- 输出：合并为单一文件，写入用户指定书籍名的父文件夹，文件名为 `该书籍名.md`。

## 脚本参考

- 使用 `scripts/merge_final.py` 完成合并。
- 默认按当前工作目录推导路径：
  - 输入：`<当前目录>/typeset-result/`
  - 输出：`<当前目录>/<书籍名>.md`（书籍名默认取当前目录名）
- 可选参数：
  - `--base-dir` 指定书籍目录
  - `--input-dir` 指定完整输入目录
  - `--book-name` 指定书籍名
  - `--output-file` 指定完整输出文件路径
  - `--base-dir-from` 使用 UTF-8 文本文件提供书籍目录（首个非空行）
  - `--input-dir-from` 使用 UTF-8 文本文件提供输入目录（首个非空行）
  - `--book-name-from` 使用 UTF-8 文本文件提供书籍名（首个非空行）
  - `--output-file-from` 使用 UTF-8 文本文件提供输出文件路径（首个非空行）

示例：

```bash
python .agent/skills/pdf-set/scripts/merge_final.py --base-dir "C:\path\to\book"
```

## 阶段 1：确认顺序
1. 读取 `typeset-result/` 内的文件列表，按文件名前的序号排序。
2. 严格按序号顺序合并；不要调整内容。

## 阶段 2：合并规则
- 仅做拼接，不做任何排版、清理、纠错或格式调整。
- 保持每个文件内容原样；必要时用换行分隔文件内容，避免黏连。

## 阶段 3：输出检查
- 输出路径正确（书籍名父文件夹内）。
- 文件名严格为 `该书籍名.md`。
- 合并顺序与文件名前序号一致。
